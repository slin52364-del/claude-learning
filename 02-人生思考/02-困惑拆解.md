# 困惑拆解

> 用途：当你被某个问题困住，理不清头绪时，用结构化的方式把它拆开看。
> 一次只处理一个困惑，写完之后往往会轻很多。

---

## 困惑描述

> 用一两句话说清楚这个困惑是什么。不用完整，先把脑子里的话倒出来。

```
我现在搞不清楚的是：
```

---

## 一、困惑的本质是什么

> 这个困惑属于哪一类？勾选或自填。

- [ ] 我不知道自己想要什么（目标不清）
- [ ] 我知道想要什么，但不知道怎么做（路径不清）
- [ ] 我知道怎么做，但做不到（执行卡住了）
- [ ] 两个都想要，但它们冲突了（价值取舍）
- [ ] 我对别人/关系感到困惑（人际）
- [ ] 其他：

---

## 二、拆解

**这件事里，我能控制的是什么？**

**我完全控制不了的是什么？**

**我在纠结什么？把两边都写下来：**

| 选项 A | 选项 B |
|--------|--------|
|        |        |
| 好处：  | 好处：  |
| 代价：  | 代价：  |

---

## 三、追问

**这个困惑让我最害怕的结果是什么？**

**如果那个结果真的发生了，然后呢？**

**五年后，这件事还重要吗？**

---

## 四、我现在真正需要的是什么

> 不是解决方案，而是：我需要的是一个答案？还是时间？还是被人听见？

```
我真正需要的是：
```

---

## 五、下一步行动（如果有的话）

> 哪怕只是一个很小的动作。

```
我接下来可以做的一件事是：
```
