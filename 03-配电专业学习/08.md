# 08. 馈线自动化（FA）——故障之后，系统自己来处理

## 从手动到自动

前几篇讲的故障处理流程是这样的：

1. 保护动作，线路跳闸
2. 故障指示器上报，调度判断故障区段
3. 调度下令，操作人员去现场拉开关
4. 故障隔离后，调度下令重新送电

这套流程的问题：**慢**。

操作人员从接令到到达现场，快则十几分钟，慢则一两个小时。这段时间里，整条线路的用户都停电。

馈线自动化（FA，Feeder Automation）要做的事：**把上面这套流程，自动完成，不用人介入。**

---

## FA 做的三件事

故障发生后，FA 系统自动完成：

**第一步：定位**
判断故障在哪个区段（跟故障指示器原理一样，但 FA 用的是开关本身上报的电流、状态信息）。

**第二步：隔离**
自动跳开故障区段两侧的开关，把故障段从线路上切出去。

**第三步：恢复**
对故障段以外的失电区域，通过合上联络开关从备用电源转供，恢复供电。

```
正常运行：
变电站A ──[开关1]──[开关2]──★故障★──[开关3]──[联络开关]──变电站B（断开）

故障发生，保护跳闸，整条线路失电。

FA 动作后：
变电站A ──[开关1]──[开关2]（断开）  ★故障段★  （断开）[开关3]──[联络开关]──变电站B（合上）

结果：
- 故障段隔离，等人工处理
- 开关3右侧的用户，从变电站B恢复供电
- 开关2左侧的用户，重合闸恢复供电
```

---

## FA 的三种实现方式

不是所有 FA 都一样，实现方式决定了速度、可靠性和成本。

### 方式一：重合器+分段器（就地型）

不依赖通信和主站，完全靠开关之间的配合逻辑自动完成。

**核心设备：**
- **重合器**：带重合闸功能的断路器，故障后自动重合几次
- **分段器**：不能自己断故障电流，但能记录"经历了几次大电流"，达到设定次数后断开

**逻辑：**

```
重合器（变电站出口）── 分段器1 ── 分段器2 ── 末端

发生故障：
1. 重合器检测到故障，跳闸
2. 重合器第一次重合：
   - 如果故障在分段器1之后，重合器又检测到故障电流，分段器1记录一次
   - 重合器再次跳闸
3. 重合器第二次重合：
   - 分段器1记录第二次，达到设定值（比如2次），分段器1断开
   - 这次重合器不再检测到故障（分段器1已隔离故障侧）
   - 重合器重合成功，分段器1以左恢复供电
   - 分段器1以右继续失电
```

**优点**：不需要通信，不依赖主站，可靠性高
**缺点**：速度慢（要经过多次重合才能完成），用户要经历多次短暂停电，复杂网络适应性差

### 方式二：集中型 FA（主站控制）

所有开关把状态、电流信息实时上传给配电主站。主站判断故障位置，下发遥控指令给开关。

```
配电主站
  │（通信）
  ├── 遥控开关1
  ├── 遥控开关2
  ├── 遥控开关3
  └── 联络开关
```

**逻辑：**

1. 故障发生，主站收到各开关的状态变化和故障电流信息
2. 主站运行 FA 算法，定位故障区段
3. 主站下发遥控：断开故障两侧开关，合上联络开关
4. 完成

**优点**：逻辑灵活，可以处理复杂网络拓扑，调度可视化好
**缺点**：依赖通信，通信中断就失效；主站算法、数据质量要求高

**典型处理时间：1-3分钟**

### 方式三：就地智能型（分布式 FA）

介于两者之间。相邻开关之间通过对等通信（不经过主站）协商，共同判断故障位置并动作。

**逻辑：**

开关之间直接通信，互相告诉对方"我这里有没有故障电流"，两个相邻开关一个有、一个没有，它们自己判断故障就在它们之间，各自执行隔离动作。

**优点**：速度快（不需要绕路通过主站），对主站通信依赖小
**缺点**：相邻开关之间的通信可靠性要求高，调试复杂

**典型处理时间：几秒到十几秒**

---

## FA 的前提条件

FA 不是装个软件就能用，它对现场设备有要求：

| 要求 | 说明 |
|------|------|
| 开关可遥控 | 普通手动开关不行，需要电动操作机构 |
| 通信覆盖 | 集中型和分布式型都需要通信 |
| 电源 | 开关需要操作电源（电池或取能） |
| 网络有联络点 | 要恢复转供，线路之间得有联络开关 |
| 数据质量 | 主站收到的信息要准确，否则判断会出错 |

实际上很多地区的配电网，线路老、设备老，改造成本高，FA 覆盖率并不高。

---

## FA 动作失败会怎样

FA 不是万能的，以下情况会失败：

- **通信中断**：集中型 FA 直接失效，退化为人工处理
- **开关拒动**：遥控指令发出，开关没动，故障无法隔离
- **故障区段判断错误**：隔离了错误的区段，健全区段被误停电
- **联络开关侧也有故障**：转供路径上也有问题，转供失败

所以 FA 动作后，调度还要确认结果是否正确，必要时人工介入纠正。

---

## 和保护的关系再理清一遍

| | 继电保护 | FA |
|--|----------|-----|
| 目的 | 切除故障，保护设备 | 缩小停电范围，恢复供电 |
| 动作时间 | 毫秒级 | 秒级到分钟级 |
| 动作方式 | 就地判断，速动 | 依赖通信和算法 |
| 结果 | 跳闸（停电范围可能大） | 隔离+恢复（最小化停电） |
| 失败后果 | 设备损坏、故障扩大 | 退化为人工处理，慢但不会更坏 |

保护先动，确保设备安全；FA 后动，缩小影响。两者不替代。

## 思考问题

1. 集中型 FA 在通信中断时完全失效，就地型 FA 不依赖通信但速度慢——你们线路现在用的是哪种，或者打算用哪种？
2. FA 动作成功后，故障段仍然没电，需要人去处理。这时候的工作流程和之前相比，变化在哪里？
3. 联络开关是 FA 实现转供的关键，但正常运行时联络开关是断开的。为什么不直接合上，把两个变电站连起来运行？
