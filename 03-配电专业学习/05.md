# 05. 从一根裸线开始——每个设备为什么会存在

## 这篇的角度

前几篇是从"已有设备"出发，讲各自的作用。

这篇反过来：从最原始的线路出发，每遇到一个问题，就加一样东西。
每样东西的出现，都是为了解决一个真实的麻烦。

---

## 第一步：最原始的线路

```
变电站 ─────────────────────────── 配变1 ── 配变2 ── 配变3
```

就一根线，从变电站拉出来，串几个配变，送电。

这能用，但问题马上来了：

---

## 问题一：负荷不在一条直线上

实际的用电户分布在一片区域里，不是整齐地排一列。

如果非要所有配变都串在一条主干线上，线路就得绕来绕去：

```
变电站 ─── 配变A ─── 配变B ─── 配变C ─── 配变D（绕了很大一圈）
```

线路长、损耗大、末端电压低，而且一旦某个方向故障，毫不相关的另一个方向也受牵连。

**解决方案：加分支线**

```
变电站
  │
  主干线
  ├──┬── 分支1 ── 配变A、B（往东）
  │  │
  │  └── 分支2 ── 配变C、D（往南）
  │
  └───── 配变E（主干末端）
```

按地理方向分叉，各管各的方向，线路更短，也更整齐。

**分支**就是从主干线上引出去的支线，本质是为了适应负荷的地理分布。

---

## 问题二：一条分支出故障，整条线都停了

有了分支之后，新问题来了：

假设分支1发生短路，保护动作，变电站断路器跳闸——整条线停电，包括分支2和主干末端的配变，全停。

但分支2 和主干没问题啊，它们为什么要跟着停？

**解决方案：在分支入口加分支开关**

```
变电站
  │
  主干线
  ├──[分支开关1]── 分支1 ── 配变A、B
  │
  ├──[分支开关2]── 分支2 ── 配变C、D
  │
  └───── 配变E
```

分支开关上配置保护。分支1故障时，分支开关1检测到故障电流，自己跳闸，切断这条分支。变电站出口的保护因为有级差配合，让分支开关先动，变电站断路器不用动。

结果：只有分支1停电，其余全部正常。

**这就是分支开关的意义**：把故障限制在这条分支里，不让它扩散到主干。

> 注意：保护配在开关上，目的是"让这个开关跳闸"，从而切除开关下游的故障线段。保护保护的是线路区段，不是开关本身。

---

## 问题三：主干线出故障，还是全停

分支开关解决了分支故障的问题，但主干线故障怎么办？

主干线故障 → 变电站出口断路器跳 → 整条线全停，包括所有分支。

主干线很长，故障可能在任何一段，全停意味着所有人都要等故障修好才能来电。

**解决方案：在主干线上加分段开关**

```
变电站
  │
  ├─[分段开关A]─ 第1段 ─[分段开关B]─ 第2段 ─[分段开关C]─ 第3段（末端）
  │               │                    │
              [分支开关]           [分支开关]
```

主干线被分成几段，每段独立。

主干线第2段故障：
- 断开分段开关A和B，把第2段隔离
- 变电站重新合闸，给第1段和分支恢复供电
- 第3段暂时停电（因为它在故障段后面，路被断了）

**分段开关的意义**：缩小主干线故障时的停电范围，从"全线停"变成"某段停"。

---

## 问题四：被隔离的段，修好之前没有来电的路

到这里，故障范围已经缩小了。但被隔离的那段，以及它后面的段，在修好之前怎么办？

唯一的路就是等修好重新送电，可能要几小时。

**解决方案：加联络开关，牵一条"备用绳"**

```
变电站A                           变电站B
  │                                   │
  ├─[分段]─ 第1段 ─[分段]─ 第2段 ─[分段]─ 第3段
                                         │
                                     [联络开关]（平时断开）
                                         │
                              变电站B 的线路末端
```

平时联络开关断开，两条线各自运行。

第2段故障、被隔离之后，合上联络开关，由变电站B那侧从另一个方向给第3段送电。

第3段不用等了，马上恢复。

**联络开关的意义**：给孤立的区段提供备用路径。这就是"手拉手"的本质。

---

## 问题五：出了故障，怎么快速知道在哪里？

前面四步，每一步都假设"已经知道故障在哪了"。

现实是：线路跳闸之后，调度只知道"某条线跳了"，不知道哪一段出的问题。

没有辅助手段就只能派人沿线巡视，10 公里的线路，巡一遍要很长时间。

**解决方案一：保护配合定位（粗定位）**

如果分段开关、分支开关上都配了保护，哪个开关跳了，就说明故障在它的下游。

调度通过远动系统看哪个开关动作了，就能把范围缩小到某段/某分支。

**解决方案二：故障指示器（精定位）**

在分段开关之间、分支线上夹故障指示器。故障电流流过的地方，指示器触发；故障点之后，没有故障电流，不触发。

最后一个"触发"的指示器和第一个"未触发"的指示器之间，就是故障点。

**保护和故障指示器的意义**：一个让线路快速停下来，一个告诉你停在哪里了。两者配合，才能完成"快速定位→隔离→转供"的完整动作。

---

## 演化路径总结

```
原始线路（一根线串到底）
  │
  ▼ 负荷地理分散
加分支线
  │
  ▼ 分支故障牵连整条线
加分支开关（+ 保护，切除本分支故障）
  │
  ▼ 主干故障还是全停
加分段开关（缩小主干故障停电范围）
  │
  ▼ 被隔离的段没有备用路
加联络开关（开辟备用路径，手拉手）
  │
  ▼ 故障后不知道哪里出了问题
加故障指示器（精确定位故障位置）
```

每一步都是被逼出来的，解决一个真实的麻烦。

没有哪个设备是"规范要求所以加的"，规范要求的背后，都是这套演化逻辑。
