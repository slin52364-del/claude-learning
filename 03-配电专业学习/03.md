# 03. 联络开关——为什么要把两条线"接"在一起

## 接着上篇的思路

上一篇我们把一条放射式线路加了分段开关，变成这样：

```
变电站
  │
  ├─ 分段开关A ─── 配变1、2
  ├─ 分段开关B ─── 配变3、4
  └─ 分段开关C ─── 配变5、6（末端）
```

加了分段开关之后，故障时可以只停某一段，其他段继续供电——**停电范围缩小了**。

但有一个根本问题没解决：

> **被隔离的那段，从哪里恢复供电？**

分段开关只能"切断"，不能"另找出路"。被隔离的那段，在故障修好之前，只能一直停着。

---

## 联络开关解决的就是这个问题

加一个联络开关，把这条线的末端，和**另一条线**连起来：

```
变电站 A                          变电站 B
  │                                   │
  ├─ 分段A ─── 配变1、2              ├─ 分段D ─── 配变7、8
  ├─ 分段B ─── 配变3、4              └─ 分段E ─── 配变9
  └─ 分段C ─── 配变5、6
                  │
              【联络开关】（平时断开）
                  │
              变电站 B 的线路末端
```

现在，**有了第二条路**。

正常运行时，联络开关是断开的，两条线各管各的。一旦某段故障：

1. 隔离故障段（断开故障两侧的分段开关）
2. 合上联络开关
3. 由变电站 B 那侧，从另一个方向给非故障段恢复送电

故障段还是要停着修，但其他段**不用等修好就能恢复供电**。

---

## 两个关键概念

**开环运行**：联络开关平时是断开的，两条线各自独立运行。

**闭环设计**：虽然平时断开，但这个联络开关存在，意味着一旦需要，可以合环转供。

所以行话叫：**"开环运行，闭环设计"**——设计上是个环，运行上是开的。

为什么平时不合上，让它闭环运行？因为合环会带来短路电流倍增、继电保护整定复杂等一堆问题，成本和风险都高，在中压配电网层面通常不值得。平时开着，需要时再合，是最经济的做法。

---

## N-1 原则，自然就出来了

加了联络开关之后，这个网络满足了一个重要的规划门槛：**N-1 原则**。

> **N-1 原则**：电网中任意一个元件（一条线路、一台变压器）故障退出运行后，通过切换操作，其余设备能在不过载的前提下，恢复对所有用户供电。

没有联络开关时：一条线故障，该线所有用户停电，无法恢复 → **不满足 N-1**。

有了联络开关之后：一条线故障，可以从另一条线转供 → **满足 N-1**。

这是城市配电网的基本门槛。城市里做网架规划，方案必须先过这一关。

---

## 转供有一个前提条件

满足 N-1 不只是"有联络开关"那么简单，还要满足：

**转供时，另一条线不能过载。**

假设线路 A 满负荷，线路 B 也满负荷，A 故障了，B 去转供 A 的负荷——B 就过载了，可靠性反而出了问题。

所以规划的时候，有一个经验原则：**每条线路的正常负载率不超过 50%**（手拉手时）或 **不超过 2/3**（三分段三联络时）。留出足够的裕量，才有转供的能力。

这个"留裕量"的要求，是配电网规划里最基础的约束之一，后续做网架优化会反复用到。

---

## 从放射式到手拉手，发生了什么变化

| 维度 | 放射式 | 加了联络开关（手拉手）|
|------|--------|---------------------|
| 电源路径 | 只有一个方向来的电 | 可以从两个方向来电 |
| 故障后能否转供 | 不能 | 能 |
| N-1 满足 | 否 | 是 |
| 建设成本 | 低 | 多一个联络开关 + 联络线段 |
| 正常运行方式 | 单辐射 | 开环（联络开关断开）|

联络开关本身不贵，但它背后代表的是：**两条线路之间存在一段联络线路**，有时这段线路才是主要的投资。

---

## 继续往下推：一个联络够用吗？

手拉手（一个联络开关）是最基本的形态。但如果线路很长、负荷很多，一个联络点可能不够：

- 故障段一旦是中间段，联络侧要绕一个很大的弯才能转供
- 转供路径太长，损耗大，末端电压可能撑不住

这就引出了**多分段多联络**的需求——不止一个联络点，也不止两段分区。
